## B - How many?
https://atcoder.jp/contests/abc214/tasks/abc214_b

<details><summary>クリックで解説を開く</summary>

<br>

$`0 ≤ S ≤ 100`$ という制約が緩いので、$`0 ≤ a ≤ S, 0 ≤ b ≤ S, 0 ≤ c ≤ S`$ としてループを回し、$`a+b+c ≤ S`$ かつ $`a × b × c ≤ T`$ を満たしていれば `ans` に1を足す、という方針でも余裕で間に合った("abc214_b 1.py")。

"abc214_b 2.py"は少し工夫をしてある。<br>
まず $`0 ≤ a ≤ S`$ でループを回す。<br>
すると $`b`$ は $`0 ≤ b ≤ S-a`$ の範囲になるので、この範囲でループを回す。<br>
最後に $`c`$ はループを回す必要はない。$`0 ≤ c ≤ \min(S-a-b, T/ab)`$ となるので、`ans` に `min(S-a-b, T//a//b) + 1` を足せば良い。ただし、`a` と `b` のどちらかが0の場合はエラーが起きる。この場合は条件分岐(if文)で別で扱って、`S-a-b+1` を足す。

公式解説にひっそりと書かれた"bonus: $`0 ≤ S, T ≤ 10^{18}`$"については考え中。

### キーワード
- 多重ループ(for文)
